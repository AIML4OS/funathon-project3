Inference

1 Load the Trained Model

This section covers how to load a trained segmentation model from the MLflow Model Registry and apply it to a Sentinel-2 image to produce land-cover predictions.

1.1 From MLflow (optional)

MLflow lets you version and compare training runs, then promote the best model to a Model Registry. Follow these steps to register your model:

  1. In the top left, select Model Training.
  2. Go to Experiments and open "funathon-2026-project3".
  3. In the Runs section, compare runs using the logged metrics and select the best-performing one.
  4. Click Register model, name it "segmentation-sentinel2-model", and confirm.
  5. Verify it appears in the Model Registry.

Create a .env file at the root of your project and fill in your MLflow credentials:

MLFLOW_TRACKING_URI=""
MLFLOW_TRACKING_USERNAME=""
MLFLOW_TRACKING_PASSWORD=""

Once registered, load the model using mlflow.pyfunc.load_model:

import os
import json
import mlflow
from dotenv import load_dotenv

load_dotenv()

model_name    = "__"  # TODO: the model name registered in MLflow
model_version = "__"  # TODO: the model version you want to load
mlflow_tracking_uri = os.getenv("MLFLOW_TRACKING_URI")

mlflow.set_tracking_uri(mlflow_tracking_uri)
model = mlflow.pyfunc.load_model(model_uri=f"models:/{model_name}/{model_version}")
TipHint

  • MLFLOW_TRACKING_URI is loaded from the .env file via load_dotenv() + os.getenv().

1.2 Retrieve model metadata

The training run stores key hyperparameters as MLflow params. Retrieve them to configure the inference pipeline consistently with how the model was trained:

run = mlflow.get_run(model.metadata.run_id)

n_bands = int(run.data.params["n_bands"])
tiles_size = int(run.data.params["tiles_size"])
augment_size = int(run.data.params["augment_size"])
module_name = run.data.params["module_name"]

normalization_mean = json.loads(
    run.data.params["normalization_mean"]
)[:n_bands]

normalization_std = [
    float(v) for v in eval(
        run.data.params["normalization_std"]
    )
][:n_bands]

print(f"n_bands={n_bands}, tiles_size={tiles_size}, augment_size={augment_size}")
print(f"mean={normalization_mean}")
print(f"std={normalization_std}")
  • tiles_size: the spatial resolution (in pixels) the model was trained on.
  • augment_size: if different from tiles_size, a resize is automatically inserted in the preprocessing pipeline.
  • normalization_mean / normalization_std: per-band statistics used to standardise the image before inference. They are sliced to [:n_bands] to match the number of bands actually used by the model.

2 Apply the Model Locally

This section shows how to run inference locally — loading the image directly from MinIO over HTTPS, running the model, and post-processing the output.

2.1 Predict on a single Sentinel-2 image

The predict() utility handles the full pipeline — loading the image, preprocessing, tiling if needed, and assembling the result:

import torch
from utils import predict

image_target = "LU000/2024/4042000_2951690_0_637.tif"

image_path = (
    "https://minio.lab.sspcloud.fr/projet-formation/"
    "diffusion/funathon/2026/project3/data/images/"
    + image_target
)

satellite_img, predictions = predict(
    images=image_path,
    model=model,
    tiles_size=tiles_size,
    augment_size=augment_size,
    n_bands=n_bands,
    normalization_mean=normalization_mean,
    normalization_std=normalization_std,
    module_name=module_name,
)

print(f"Mask shape    : {predictions.shape}")
print(f"Classes found : {set(predictions.flatten().tolist())}")

Internally, predict():

  1. Opens the image from HTTPS via /vsicurl/ (no local download needed).
  2. Normalises and resizes the array with albumentations.
  3. Splits the image into tiles of tiles_size × tiles_size if the image is larger, then reassembles predictions with make_mosaic.

torch.argmax(..., dim=0) then collapses the class dimension by picking the class with the highest score at each pixel, producing a 2D integer array of shape (H, W) with values from 1 to 10 (CLC+ class IDs).

2.3 Convert predictions to polygons

create_geojson_from_mask() vectorises the class mask into a GeoDataFrame of polygons. Internally it writes the mask to a temporary GeoTIFF to preserve the georeference, then calls rasterio.features.shapes to extract contiguous regions of identical class. Each row in the output GeoDataFrame has a geometry (polygon) and a label (integer class ID from 1 to 10). Pixels with class value 0 (background / no-data) are automatically excluded.

The polygons are saved to a Parquet file whose name is derived from the image path, making it easy to match predictions to their source tile later.

3 Inference via API

Running inference locally works well for a single tile, but scaling to an entire NUTS3 region (which may contain dozens of tiles) requires more infrastructure. A REST API has been deployed for this purpose.

3.1 Architecture

The API is built with FastAPI and deployed on the SSP Cloud Kubernetes cluster. It wraps the same predict() / create_geojson_from_mask() pipeline used locally, but adds two important features:

  • S3 cache layer: predictions are stored in S3 as .npy files after the first inference call. Subsequent requests for the same image load the cached result instead of re-running the model, which drastically reduces latency for repeated or batched queries.
  • Batch processing: the /predict_nuts endpoint automatically discovers all .tif tiles for a given NUTS3/year pair in S3, runs (or loads from cache) predictions for each one in parallel, and merges the results into a single GeoJSON response.

3.2 Endpoints

The API exposes three endpoints:

Endpoint Method Description
GET /find_image Returns the filename of the Sentinel-2 tile that contains a given GPS point
GET /predict_image Runs inference on a single image; optionally returns vectorised polygons
GET /predict_nuts Runs inference on all tiles for a NUTS3/year pair and returns merged polygons

3.2.1 GET /find_image

This endpoint takes a GPS point (latitude, longitude in WGS84) and a NUTS3 identifier + year. It loads a pre-computed filename2bbox.parquet index from MinIO — a table that maps each tile filename to its bounding box in EPSG:3035 — then performs a spatial query to find which tile contains the point.

Parameters:

Name Type Description
gps_point List[float, float] Latitude, Longitude in WGS84 (EPSG:4326)
nuts_id str NUTS3 region identifier, e.g. "LU000"
year int Year of the satellite images (2018–2024)

Response: a string of the image filename containing the point.

3.2.2 GET /predict_image

This endpoint runs the full segmentation pipeline on a single Sentinel-2 tile identified by its path in S3. If a cached prediction exists for this image, it is loaded directly without running the model again.

Parameters:

Name Type Description
image str S3 path to the .tif file
polygons bool If True, also vectorise the mask and return polygons

Response: a JSON object with:

  • "mask": the 2D class mask as a nested list (H × W)
  • "polygons" (optional): a GeoJSON string containing the vectorised polygons

3.2.3 GET /predict_nuts

This endpoint discovers all .tif tiles for a NUTS3/year pair in S3, runs (or loads from cache) predictions for each one, applies produce_mask() to convert logits to class IDs, then concatenates the GeoDataFrames from create_geojson_from_mask() into a single response. This is the main entry point for region-level analysis.

Parameters:

Name Type Description
nuts_id str NUTS3 region identifier
year int Year of the satellite images (2018–2024)

Response: a JSON object with:

  • "predictions": a GeoJSON string (FeatureCollection) containing all predicted polygons for the NUTS3 region, in EPSG:3035
Note

The first call to /predict_nuts for a new NUTS3/year combination can take several minutes, as it needs to run inference on each tile. Subsequent calls are fast thanks to the S3 cache.

3.5 Exercise 6 - Display predictions on an interactive map

Folium renders interactive web maps directly in a Jupyter notebook. The satellite image is overlaid as a semi-transparent raster layer using ImageOverlay, while the predicted polygons are added as a GeoJson layer with per-class colouring.
